# Lynx on HarmonyOS：原生体验的深度融合与技术实践

<BlogAvatar list={['baiqiang', 'lynx']} />

![lynx-harmony-background](https://lf-lynx.tiktok-cdns.com/obj/lynx-artifacts-oss-sg/lynx-website/assets/blog/lynx-harmony/background.png)

随着 HarmonyOS NEXT 的发布，一个纯净的鸿蒙生态正加速到来。
对于每一位移动开发者而言，这不仅是一次前所未有的机遇，更是一场必须面对的技术变革。
我们该如何管理多端（iOS/Android/Harmony）代码，并且在新平台下可以高效复用现有业务逻辑？又该如何确保应用在鸿蒙上也能拥有丝滑流畅的原生体验？

这些问题，也是 Lynx 团队在过去一段时间中全力以赴解决的核心问题。
今天，我们激动地宣布，Lynx 已经完成对 HarmonyOS 的深度适配，并正式开源；Lynx 为开发者提供了一套完整、高效的跨端开发解决方案，无缝衔接现有业务，并充分利用 HarmonyOS 带来的独特优势。

## 为什么选择 Lynx on HarmonyOS？

Lynx 的核心目标在于使开发者能够运用其熟悉的 Web 技术栈，构建出具备原生级体验的应用程序。通过与 HarmonyOS 实现深度融合，我们已成功将 Lynx 的高性能渲染引擎与 HarmonyOS 的原生 UI 能力有机结合，由此为开发者带来以下核心优势：

- **极致性能，超越以往**：通过与 [ArkUI](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkui-overview) 的 [C API](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ndk-build-ui-overview) 深度整合，Lynx 的渲染指令可以绕开 [ArkTS](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-overview) 虚拟机的执行开销，实现接近甚至超越原生 ArkTS 的渲染性能。
- **统一代码库，多端覆盖**：一套代码，多端运行。开发者可以使用同一份代码库，同时构建 Web、iOS、Android 和 HarmonyOS 应用，极大地提升了开发效率和代码可维护性。
- **无缝的 API 访问**：通过我们的原生模块扩展能力，开发者可以轻松地在 JavaScript 中调用鸿蒙平台的原生 API，无论是系统级的服务还是自定义的 Native 模块，都可以无缝集成。更多可了解[原生模块的使用指南](/zh/guide/use-native-modules.html#platform=harmony)。
- **灵活的元件定制能力**：Lynx 不仅提供了丰富的内置元件，还允许开发者将自己开发的鸿蒙原生组件（通过 ArkTS 实现）封装并注册到 Lynx 运行时中。这意味着开发者可以根据业务需求，无限扩展应用的 UI 能力，实现高度定制化的界面和交互，同时保持原生性能。更多可了解[自定义元件的使用指南](/zh/guide/custom-native-component.html#platform=harmony)。

## 架构深度解析：当 Lynx 遇见 HarmonyOS

为了在鸿蒙上实现最佳性能，Lynx 不仅是简单地”适配”，而是进行了一次深度的架构融合。我们的方案核心在于将 Lynx 的内核与鸿蒙的 ArkUI 渲染体系直接对接。

![lynx-harmony-background](https://lf-lynx.tiktok-cdns.com/obj/lynx-artifacts-oss-sg/lynx-website/assets/blog/lynx-harmony/lynx-harmony.png)

### 渲染引擎：原生与自定义的无缝融合

Lynx 提供了卓越的渲染性能，确保应用在各种复杂场景下依然流畅运行。开发者既能享受 Lynx 带来的极致性能，又能随时利用 ArkTS 的丰富生态与能力。

- **原生组件渲染**：Lynx 的 UI 树直接映射为鸿蒙的原生组件。我们利用了 ArkUI 最新的 **C API**，将 DOM 操作直接转换为对原生 UI 节点的调用，尤其是在复杂和高频的 UI 更新场景下，性能优势尤为明显。

- **自定义元件能力**：通过 [`BuilderNode`](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-user-defined-arktsnode-buildernode) 和 [`Builder`](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/arkts-builder) 函数动态构建 ArkTS 子树，可以无缝嵌入 Lynx 创建的 ArkUI 节点树。

### LynxView：高效易用的接入层

将响应式的 ArkTS 与 Lynx 客户端能力深度结合，设计了 `LynxView` 组件化 API 体系，为开发者提供了一种高效、易用的 Lynx 接入方式。

<CodeFold>
```ts title=EntryAbility.ets
import { LynxServiceCenter, LynxEnv, LynxServiceType } from '@lynx/lynx';
import { LynxDevToolService } from '@lynx/lynx_devtool_service';

export default class EntryAbility extends UIAbility {
  onCreate(want: Want, launchParam: AbilityConstant.LaunchParam): void {
    LynxServiceCenter.registerService(LynxServiceType.DevTool, LynxDevToolService.instance);
    // ...
    LynxEnv.initialize(this.context);
    // ...
  }
}
```
</CodeFold>

使用 `LynxView` 加载 `bundle` 即可：

<CodeFold>
```ts title=Index.ets {12-17}
import { LynxServiceCenter, LynxEnv, LynxServiceType, LynxTemplateResourceFetcher, LynxMediaResourceFetcher, LynxGenericResourceFetcher, LynxView, } from '@lynx/lynx';
import { LynxDevToolService } from '@lynx/lynx_devtool_service';

@Entry
@Component
struct Index {
  // ...
  private url: string = 'your bundle file';

  build() {
    Column() {
      LynxView({
        templateResourceFetcher: this.templateResourceFetcher,
        mediaResourceFetcher: this.mediaResourceFetcher,
        genericResourceFetcher: this.genericResourceFetcher,
        url: this.url,
      }).width('100%').height('100%');
    }
    .size({ width: '100%', height: '100%' })
  }
}
```
</CodeFold>

- **[LynxView](/zh/api/lynx-native-api/lynx-view/load-template.html#harmony)**：它是一个 ArkTS 自定义组件，它遵循 ArkTS 的生命周期并且自动处理了所有关于模板加载、内存管理和组件复用的繁杂事务。开发者只需传入需要的展示 bundle 路径就能自动完成渲染。

- **[LynxResourceFetcher](/zh/api/lynx-native-api/lynx-generic-resource-fetcher/fetch-resource.html#harmony)**：它提供了灵活的资源加载能力，开发者只需实现该协议，即可将 App 已有的资源加载能力无缝对接给 Lynx。确保了 Lynx 可以轻量、高效地融入现有的开发体系中。

- **[LynxService](/zh/api/lynx-native-api/lynx-service.html#2-扩展-lynx-service)**：灵活的能力注入机制。提供一些宿主应用特性强相关的能力例如网络请求，日志服务等，可以在 App 中注入自定义实现，在运行时 Lynx 会通过服务查找机制查找注册的 Service，并通过 `LynxService` API 调用。

### 事件体系：精妙协同，驾驭复杂交互

我们构建了一套完整的事件处理机制，确保在复杂的交互场景下，手势和事件能够被精准地捕获和响应。

- **自定义 HitTest**：在 Lynx 的 UI 树上精确查找响应事件的节点。
- **手势识别**：无缝集成了鸿蒙系统的手势识别能力，如 `tap`、`longpress` 等。
- **场景控制**：提供了丰富的 API，如 `user-interaction-enabled` 和 `block-native-event`，允许开发者精细化控制事件的穿透和消费逻辑，完美处理与原生页面的混合滚动等复杂场景。


## 开启您的 Lynx on HarmonyOS 之旅

我们相信，Lynx 与 HarmonyOS 的深度融合，将为开发者在新的生态中提供强大的生产力。我们已经准备好了详细的开发文档和示例代码，帮助您快速上手。

### 使用预构建的 Lynx Explorer

若您希望以最便捷的方式开启开发，使用预构建的 Lynx Explorer 是绝佳之选。您可以根据按照下面的步骤操作安装 hap 包，轻松迈出开发的第一步。

1. **安装 DevEco Studio**

从 [官网](https://developer.huawei.com/consumer/cn/deveco-studio/) 下载安装最新的 DevEco Studio。

2. **下载 Lynx Explorer**

下载预编译应用 [`lynx_explorer-default-unsigned.hap`](https://github.com/lynx-family/lynx/releases/latest/download/lynx_explorer-default-unsigned.hap)。

3. **在模拟器上安装 Lynx Explorer**

```bash
export PATH=/Applications/DevEco-Studio.app/Contents/sdk/default/openharmony/toolchains:$PATH
hdc install lynx_explorer-default-unsigned.hap
```

4. **开始开发**

在下面的二维码选项点击“复制链接”按钮并粘贴到 Lynx Explorer 的 Card URL 输入框，然后点击 "Go"。
